# Planning with Files
Activates when starting a new task or a complex feature in the **TalentSphere** project.

## Instructions
1. Create a structural blueprint before writing any code.
2. Define file dependencies and required modules across frontend and backends.
3. List implementation steps in a logical order to minimize build breakage.
4. Update the `SSOT` (Single Source of Truth) if architectural changes are proposed.

## Constraints
- Focus only on the impacted services.
- Always verify dependencies first before suggesting service changes.
