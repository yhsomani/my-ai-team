## 2026-05-08 - [Critical] Insecure AWS Secrets Manager Authentication Fixed
**Vulnerability:** Found in `TalentSphere-Unified/services/shared/src/main/java/com/talentsphere/shared/secrets/SecretsManagerConfig.java`. AWS credentials (`AWS_ACCESS_KEY_ID` and `AWS_SECRET_ACCESS_KEY`) were being read from environment variables, concatenated, Base64-encoded, and sent via a Basic Authentication HTTP header to a manual Secrets Manager API endpoint. This is a critical security vulnerability that exposes root/IAM credentials in plain-text/Base64 over the network.
**Learning:** Manual API calls were incorrectly using HTTP Basic Auth for AWS, which actually requires Signature V4. The vulnerability actively leaked sensitive credentials on every secret-fetch attempt. Additionally, the AWS SDK was missing from the service's `pom.xml`, leading to this dangerous workaround.
**Prevention:** Always use the official AWS SDK (`software.amazon.awssdk:secretsmanager`) for AWS integrations. The SDK inherently handles secure SigV4 signing. Never construct manual HTTP requests with AWS secret keys, and never transmit AWS keys in HTTP Basic Auth headers.
